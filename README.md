What are the Advantages and Disadvantages of White Box Testing?

It checks each branch and each statement in the code.
It helps to find errors at an early stage.
It removes unnecessary or extra lines of code (code optimization).
It helps to identify runtime errors and exception handling issues.
It does not require a user interface for testing.
It helps to detect complex logic errors in the program.

Disadvantages 

The tester must have programming knowledge.
It focuses only on internal code and may miss external issues.
It can produce false-positive results.
It requires deep knowledge of coding and system architecture.
It is time-consuming for large applications.
It cannot identify missing functionalities or customer requirements.

What is Statement Coverage and Path Coverage?
1. Statement Coverage

Statement Coverage is a White Box Testing technique used to verify that every statement or line of code in a program is executed at least once during testing. The main purpose of statement coverage is to check whether all the statements in the program have been executed or not. It helps testers identify defects in the program logic and ensures that every part of the code is tested. In statement coverage, test cases are designed in such a way that all statements in the source code are covered during execution.

2. Path Coverage

Path Coverage is a White Box Testing technique used to verify all possible execution paths in a program. It ensures that every independent path from the start to the end of the program is tested at least once. The main objective of path coverage is to identify errors in the program logic and control flow. In this technique, testers analyze different paths in the program and create test cases to execute each path. Path coverage provides a more thorough testing approach by checking all possible execution flows of the application.
